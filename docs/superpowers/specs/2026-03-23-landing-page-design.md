# ResumeAI Landing Page Design Spec

## Overview

A standalone Next.js marketing site deployed at `landing-resume-ai.johnathanwwh.com`. All CTAs link to the app at `https://resume-ai.johnathanwwh.com`. The page is statically exported for maximum SEO performance and serves as the primary entry point for new users discovering ResumeAI.

---

## Goals

- Entice job seekers to sign up for ResumeAI
- Communicate all major features clearly
- Rank well in search engines (SEO-optimized)
- Retain the same color scheme as the app
- Deploy independently from the React client app

---

## Tech Stack

| Concern | Choice | Reason |
|---|---|---|
| Framework | Next.js 15+ (App Router) | Static export, SEO-friendly, file-based routing |

## Project Structure

**Monorepo placement:** `landing/` lives at the monorepo root but is NOT added to the existing npm workspaces (`client/`, `server/`). It is a standalone project тАФ run `npm install` and `npm run dev` independently from within `landing/`.

**Client vs. Server Components:** Components that use Framer Motion hooks (`motion`, `useReducedMotion`, `useInView`) or browser APIs must include `'use client'` at the top of the file. Server Components: `layout.tsx`, `page.tsx`, `Footer.tsx`. Client Components: `Hero.tsx`, `WhySection.tsx`, `FeaturesSection.tsx`, `HowItWorks.tsx`, `TemplatesSection.tsx`, `Navbar.tsx`.

**tsconfig.json:** Use standard Next.js 15+ defaults as generated by `create-next-app` тАФ no modifications needed.

**CSS tokens:** The Color Tokens table is informational. Use Tailwind utility classes directly in components (e.g., `bg-gray-900`, `text-blue-500`) rather than declaring CSS custom properties. `globals.css` only needs Tailwind's `@tailwind base/components/utilities` directives.

```
landing/                          # standalone Next.js app, not in npm workspaces
  public/
    og-image.png                  # 1200├Ч630 static OG image: dark bg, ResumeAI wordmark + tagline
  app/
    layout.tsx                    # Server Component: root layout, font, metadata, globals
    page.tsx                      # Server Component: home page тАФ composes all section components
    globals.css                   # Tailwind base directives only (no CSS custom properties)
    sitemap.ts                    # auto-generated sitemap for SEO
    robots.ts                     # robots.txt for SEO
  components/
    Navbar.tsx                    # 'use client' тАФ sticky nav, logo left, CTA button with whileHover animation
    Hero.tsx                      # 'use client' тАФ dark hero, blobs, Framer Motion entrance
    WhySection.tsx                # 'use client' тАФ 3 pain-point cards, scroll animations
    FeaturesSection.tsx           # 'use client' тАФ bento grid, scroll animations
    HowItWorks.tsx                # 'use client' тАФ 3-step flow, id="how-it-works" for scroll anchor
    TemplatesSection.tsx          # 'use client' тАФ abstract template cards, scroll animations
    Footer.tsx                    # Server Component тАФ static logo + links
  package.json
  next.config.ts                  # output: 'export', images: { unoptimized: true }
  tailwind.config.ts
  tsconfig.json
```

3. **Track, prep, and apply** тАФ Log your application, generate interview questions, practice your answers, and stay organized until you land the offer.

## Animation Strategy (Framer Motion)

| Element | Animation |
|---|---|
| Hero headline + subheadline | Fade + slide up on mount, staggered 0.15s |
| Hero blobs | Slow scale pulse loop (6s) |
| Why/Features/HowItWorks cards | Fade + slide up on scroll enter (`whileInView`) |
| Template cards | Fade in on scroll, staggered |
| CTA buttons | Scale on hover (`whileHover={{ scale: 1.03 }}`) |

All animations respect `prefers-reduced-motion` via Framer Motion's `useReducedMotion()`.
